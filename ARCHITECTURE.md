## Вариант для слайда (коротко)

### Технологии и архитектура

**Сегодня (MVP):**
- iOS-приложение на **SwiftUI** + архитектура **MVVM**
- Кроссплатформенный масштаб: переиспользуемые компоненты и дизайн-система

**Целевая архитектура экосистемы:**

```
   Пациент
   📱 iOS / Web        ☎ Телефон · WhatsApp · Telegram
        │                         │
        └──────────┬──────────────┘
                   ▼
        ┌─────────────────────┐
        │   API Gateway       │  ← единая точка входа
        └─────────┬───────────┘
                  ▼
     ┌─────────────────────────────┐
     │      AI-ядро (LLM + голос)   │  STT / NLU / TTS
     │  распознавание · намерение   │
     └─────┬──────────────────┬─────┘
           ▼                  ▼
   Маршрутизация        База знаний / RAG
   к специалисту        (мед. протоколы)
           │
   ┌───────┴────────┬───────────┬─────────────┐
   ▼                ▼           ▼             ▼
 Запись/        Телемедицина  Напоминания   Аналитика
 расписание     (WebRTC)      (Push)        (BI/отчёты)
   │
   ▼
 Интеграции: МИС клиник · ОСМС/страховые · фарма
```

**Принципы:** микросервисы · API-first · безопасность мед. данных · 24/7 ·
тысячи одновременных обращений.

## Спикерские заметки (что говорить под слайд)

> «Архитектурно мы строим **API-first экосистему**, а не одно приложение.
> Сейчас готов **iOS MVP на SwiftUI с чистой MVVM** — это витрина продукта и
> каркас под быстрое развитие.
>
> Ядро системы — **AI-оператор**: многоканальный вход (телефон, мессенджеры,
> приложение) → распознавание речи и намерения → ответ или **маршрутизация**
> к нужному специалисту. Вокруг ядра — независимые сервисы: запись,
> телемедицина, напоминания, аналитика.
>
> Всё спроектировано под **24/7 и тысячи одновременных обращений** со
> снижением нагрузки на персонал. Дальше — интеграции с медицинскими
> информационными системами, страховыми и фармой».

## Почему такой выбор (если спросят)

- **SwiftUI + MVVM** — быстрая разработка, нативный UX уровня Apple Health,
  лёгкая поддержка и тестируемость.
- **Микросервисы + API Gateway** — каждый модуль (AI, телемед, напоминания)
  масштабируется независимо.
- **AI-ядро отдельным слоем** — можно менять модель/провайдера без переписи
  клиентов; добавляются новые каналы без изменения бизнес-логики.
- **Безопасность данных** — мед. данные шифруются, доступ по ролям; готовность
  к требованиям регулятора (Минздрав РК).
